## 2. MQTT-over-WebSocket client (JS shim + PyScript wrapper)

- [x] 2.1 Create `heart-message-manager/static/mqtt_ws_client.js` exporting `createMqttWsClient({ url, username, password, topic, longDisconnectMs, onEnvelope, onStatus })` — opens a WebSocket to the broker, performs the MQTT 3.1.1 CONNECT frame (with username/password), SUBSCRIBEs to the topic, decodes PUBLISH payloads as UTF-8 JSON and calls `onEnvelope(rawString)`, surfaces `connected` / `reconnecting` / `paused` / `error` to `onStatus`
- [x] 2.2 Add auto-reconnect with exponential backoff (1s → 2s → 4s → 8s → 16s → 32s → 60s cap) on close or error
- [x] 2.3 Track `lastConnectedAt` internally. After each `connected` event, start a timer; when the timer exceeds `longDisconnectMs` (default 300000 ms), emit `paused` to `onStatus` with `{ elapsedMs }`. The timer is reset on each `connected` event. Do NOT install a `visibilitychange` listener — the elapsed-time timer covers all paths to a long disconnect.
- [x] 2.4 On successful reconnect after a `paused` window, emit `connected` to `onStatus` with `{ wasLongDisconnect: true, elapsedMs }`. The base template's `app.js` listens for this flag and triggers the wipe + re-seed sequence. On short reconnects, the `connected` event SHALL NOT carry `wasLongDisconnect: true`.
- [x] 2.5 Create `heart-message-manager/static/mqtt_ws_client.py` exposing a `MqttWsClient` Python class that calls the JS shim via `create_proxy`; the Python side's `_on_envelope` is a user-supplied callable
- [x] 2.6 Add `tests/test_mqtt_ws_client_py.py` covering the Python wrapper's surface (no `MqttWsClient` instantiation in CI without a JS runtime — keep this as a smoke test that imports the module and checks the class signature; the JS shim is exercised manually in the browser)
