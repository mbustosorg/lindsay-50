- [ ] 1.2 Add `tests/test_sign_status.py` with round-trip tests: `update()` stores the dict under the lock, `snapshot()` returns a defensive copy (mutating the returned dict does not affect the store), `age_seconds()` returns the right number with a fake monotonic clock, `state()` returns `"live"` / `"unsure"` / `"offline"` at the right thresholds, and `snapshot()` on an empty store returns `None` (and `state()` returns `"offline"`). Cover the concurrent-update case (two threads call `update()` simultaneously — the lock prevents torn reads).
